= Release Notes

This lab project ist yet to yield a first release. Stay tuned!

== Versioning

The `go/` workspace contains four independently versioned **products** and a set of
internal **libraries**.

Products (`zero-epa`, `zero-pdp`, `zero-caddy`, `ti` CLI) are versioned independently.
The version numbers live as variables in `go/Justfile`
(`ZERO_EPA_VERSION`, `ZERO_PDP_VERSION`, `ZERO_CADDY_VERSION`, `TI_CLI_VERSION`) and are
the single source of truth. They are injected into the binaries at build time via
`-ldflags "-X <pkg>.Version=<v>"` — `just build-<product>` for local builds and the
`--build-arg VERSION=...` passed by `just docker-build-<product>` for images. The
in-source `Version` symbols default to `"dev"` for plain `go build`.

A release is marked with a per-product, annotated git tag created from the Justfile var:
`zero-epa/vX.Y.Z`, `zero-pdp/vX.Y.Z`, `zero-caddy/vX.Y.Z`, `ti/vX.Y.Z`
(`just tag-<product>`, or `just tag` for all four; `just push-tags` to publish). These are
release markers only — they are deliberately not Go module version tags.

The library modules (brainpool, gempki, oauth, oidf, dpop, pep, nonce, kms, kon, pkcs12,
gemidp, asl, bff) are **not** versioned or tagged. They are resolved locally through the
tracked `go.work` workspace, so the commit pseudo-versions in their `go.mod` `require`
blocks are cosmetic. Run `go work sync` (and `just tidy`) to keep them aligned; never
hand-edit those pseudo-versions.
