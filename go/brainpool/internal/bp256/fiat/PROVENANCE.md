# Provenance: brainpoolP256r1 field arithmetic

`bp256_fiat64.go` is **generated, machine-checked code** emitted by
[Fiat Cryptography](https://github.com/mit-plv/fiat-crypto). It is committed
verbatim and must never be hand-edited. `bp256_invert.go` is the field inversion
addition chain emitted by [addchain](https://github.com/mmcloughlin/addchain).
`bp256.go` is the hand-written `Element` wrapper (curve-independent, adapted from
the Go standard library `crypto/internal/fips140/nistec/fiat` template).

## Why this is the "formally verified" layer

Fiat Cryptography synthesizes field arithmetic from a Coq proof that the emitted
code computes the correct modular result for every input within the documented
bounds. This is the same provenance the Go standard library relies on for its
P-224/P-256/P-384/P-521 field elements. We reuse the **identical toolchain,
image, and flags** the standard library uses, changing only the curve prime.

## Field prime

brainpoolP256r1, RFC 5639 §3.4:

    p = 0xA9FB57DBA1EEA9BC3E660A909D838D726E3BF623D52620282013481D1F6E5377

The prime has no special reduction structure (unlike the NIST Solinas primes), so
the general **word-by-word Montgomery** synthesizer is the correct choice, with 4
limbs of 64 bits.

## Reproducing `bp256_fiat64.go`

The generator runs inside the pinned `fiat-crypto:v0.0.9` image — the exact image
and commit the Go standard library pins (see
`$GOROOT/src/crypto/internal/fips140/nistec/fiat/Dockerfile`):

```dockerfile
FROM coqorg/coq:8.13.2
RUN git clone https://github.com/mit-plv/fiat-crypto && cd fiat-crypto && \
    git checkout 23d2dbc4ab897d14bde4404f70cd6991635f9c01 && \
    git submodule update --init --recursive
RUN cd fiat-crypto && eval $(opam env) && make -j4 standalone-ocaml SKIP_BEDROCK2=1
ENV PATH /home/coq/fiat-crypto/src/ExtractionOCaml:$PATH
```

Build it once, then run the driver in this directory:

```sh
docker build --platform linux/amd64 -t fiat-crypto:v0.0.9 .
go install github.com/mmcloughlin/addchain/cmd/addchain@v0.4.0
go run generate.go
```

- fiat-crypto: commit `23d2dbc4ab897d14bde4404f70cd6991635f9c01` (release v0.0.9)
- addchain: v0.4.0
- machine word size: 64
- requested operations: `mul square add sub one from_montgomery to_montgomery selectznz to_bytes from_bytes`

The exact `word_by_word_montgomery` invocation is recorded both in this file and
in the `Autogenerated:` header comment at the top of `bp256_fiat64.go`.
